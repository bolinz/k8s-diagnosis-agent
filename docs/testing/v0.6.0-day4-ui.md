# v0.6.0 Day 4 UI Validation

Day 4 target: expose diagnosis trust/explainability fields in the Workbench detail UI.

## Scope

- `Diagnosis Quality` panel
- `Uncertainties` panel
- `Evidence Attribution` panel

## Automated Checks

```bash
cd web
npm run test
npm run build

cd ..
python3 -m pytest -q tests/test_http_ui.py
```

## Manual Checks

1. Open the UI and select any report with populated `qualityScore`.
2. Verify `Diagnosis Quality` shows:
   - overall percentage
   - method
   - fallback flag
   - dimension rows
3. Verify `Uncertainties` list shows when data exists and shows empty-state text when absent.
4. Switch detail view to `attribution` and verify `Evidence Attribution` rows display:
   - source
   - signal
   - optional tool/object/time hints
5. Select a legacy report (without v0.6 fields) and confirm page does not blank/crash.

## Pass Criteria

- No runtime errors during view switching.
- No blank detail page for legacy reports.
- All three Day 4 panels visible with correct empty-state handling.
