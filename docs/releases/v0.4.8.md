# v0.4.8 Release Notes

`v0.4.8` is a reliability-focused patch release.

## Highlights

- `POST /alert` is now asynchronous and returns `202` quickly with `requestId`.
- New `GET /api/alerts/{requestId}` endpoint for task status and result lookup.
- HTTP response writes now handle client disconnects (`BrokenPipeError`/`ConnectionResetError`) without noisy traceback logs.

## Included Changes

1. Async alert execution
- Added in-memory alert task manager with background worker threads.
- Alert task status transitions: `queued -> running -> succeeded|failed`.
- Task payload includes `createdAt`, `startedAt`, `finishedAt`, `reportName`, and error/result summary.

2. HTTP robustness improvement
- Response write path now catches client disconnect exceptions and emits structured warning logs.
- Prevents repeated traceback noise in pod logs when callers timeout before server responds.

3. Version updates
- Python package: `0.4.8`
- Helm chart/appVersion: `0.4.8`
- Diagnosis report analysis version: `0.4.8`
