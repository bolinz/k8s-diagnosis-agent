# v0.5.0 Release Notes (Draft)

`v0.5.0` is a major UI/operability release for the embedded diagnosis workbench.

## Highlights

- Timeline navigator usability improvements for dense event streams.
- Better keyboard-driven operation for on-call workflows.
- Stronger UI regression coverage for timeline/relation rendering behaviors.

## Included Changes

1. Timeline navigation and density cues
- Added event density strip in timeline panel for quick jump into dense buckets.
- Added focused event quick navigation (`Prev` / `Next` controls).
- Added keyboard shortcuts for focused timeline traversal:
  - `,` previous event
  - `.` next event

2. Event navigator readability improvements
- Grouped event navigator by signal.
- Added group collapse/expand behavior to reduce noise.
- Added group sort modes:
  - `By Count`
  - `By Time`
- Active/focused event group is highlighted and auto-expanded.

3. Workbench operator ergonomics
- Added built-in shortcut help panel (`Shortcuts` button and `?` toggle).
- Persisted timeline group sort preference in local storage.
- Kept two-pane layout and relation graph overlap guards with regression assertions.

4. Test and quality updates
- Extended Playwright e2e coverage for:
  - density strip rendering
  - grouped navigator collapse behavior
  - timeline keyboard navigation
  - shortcut help visibility
  - timeline group sort persistence after reload
- Existing relation graph anti-overlap and pane-height consistency checks remain enforced.

## Validation Checklist

- Backend:
  - `python3 -m pytest -q`
  - `PYTHONPYCACHEPREFIX=/tmp/pythoncache python3 -m compileall agent tests`
- Frontend:
  - `cd web && npm run test -- --run`
  - `cd web && npm run test:e2e:with-server`
- Manual smoke:
  - open `/?report=preview-4`
  - verify `Shortcuts` panel opens/closes via button and `?`
  - verify timeline navigator group sort can switch and persists after reload
  - verify `,` and `.` move timeline focus when neighboring events exist
