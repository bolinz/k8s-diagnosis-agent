# v0.4.7 Release Notes (Draft)

This document is the draft release body for GitHub Release `v0.4.7`.

## Highlights

- Controlled autonomous probing safety remains bounded and test-covered.
- Root-cause attribution is now ranked deterministically (`score` + `rankReasons`).
- Event bursts are collapsed into a single aggregated diagnosis report.
- Each diagnosis now carries an audit trace (`traceId`) for operations debugging.
- UI detail view better surfaces root candidate and earliest abnormal signal.

## Included Changes

1. Attribution scoring v1
- Deterministic scoring and tie-breaking for `rootCauseCandidates`.
- Scores are stored with ranking reasons to improve operator explainability.

2. Event-storm suppression v1
- Windowed object/symptom aggregation for event-triggered diagnosis.
- Burst threshold is configurable via:
  - `K8S_DIAGNOSIS_EVENT_STORM_THRESHOLD`
- Aggregated burst report uses fallback path to avoid unbounded model requests.

3. Probing audit trail v1
- Added trace metadata including:
  - `traceId`
  - tool sequence and per-tool duration
  - scope guard hit count
  - fallback reason
- Trace is surfaced in report status (`diagnosisTrace`) and `modelInfo.traceId`.

4. UI attribution readability
- Added `Top Root Candidate` section.
- Added list-level root-candidate count chip.
- Emphasized first abnormal signal in `Evidence Timeline`.

## Deployment Notes

- Recommended image tag for rollout:
  - `ghcr.io/bolinz/k8s-diagnosis-agent-app:sha-<full-commit-sha>`
- Optional new env var:
  - `K8S_DIAGNOSIS_EVENT_STORM_THRESHOLD` (default `5`)

- `python3 -m pytest -q`
- `PYTHONPYCACHEPREFIX=/tmp/pythoncache python3 -m compileall agent tests`
- Confirm CI green on `main`.
- Controlled cluster check:

  - burst events create at most one aggregated report in-window
